# Kajabi Product Design Principles

## 1. Zoom out before you zoom in.

Before solving the screen, understand the system. How does this connect to everything else in Kajabi? Is there a bigger issue this project is touching? How far can existing patterns take us? Start with the whole, then work your way to the part.

**Failure modes:**
- Creating a new page layout or workflow when an existing page template or pattern would solve the problem
- Solving for one part of the app or one screen without considering how it connects to the rest of that domain or the system itself
- Missing a larger systemic issue because the focus was too narrow on the immediate project
- Building a feature that overlaps with or competes with an existing feature without reconciling the relationship (e.g., two visual builders for similar jobs, duplicate blueprint patterns, similar UI flows in different parts of the app)

**Overlapping features check:** When reviewing, always ask: Are there existing features in Kajabi that solve a similar problem or share similar UI patterns? What makes this feature distinct from those? How much overlap exists, and what is the current thinking to address it? If this isn't clear, flag it. One-off features that duplicate existing capabilities create confusion for Heroes and maintenance burden for the team.

## 2. All in one, not all at once.

Show users what's essential to move forward, then let them go deeper if they want to. Resist the urge to over-design or add nonessential details. The goal is to solve a user's problem, elevate their expertise, and let the design fade into the background.

**Failure modes:**
- Showing all options and settings at once instead of progressive disclosure
- Adding decorative elements or unnecessary detail that doesn't serve the user's goal
- Design that draws attention to itself rather than fading into the background

## 3. Sweat the details.

Hierarchy, typography, color, spacing, layout, interactions, animations. These fundamentals are baked into our design system, but we don't rely on it to do all the thinking for us. Our mastery of these skills and high bar for quality allows us to craft bespoke and memorable moments when the work calls for it.

**Failure modes:**
- Inconsistent spacing, typography, or color usage that doesn't follow the system
- Weak visual hierarchy that makes it hard to scan or know what I'm supposed to do on a page or what will happen next in the flow
- Interactions or animations that feel off or unpolished
